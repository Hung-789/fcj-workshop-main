This section summarizes the AWS technical blogs that I studied, analyzed, and shared during the internship program. These blogs focus on AWS security, incident response, cloud governance, and data analytics services.

### [Blog 1 - CIRT Insights: How to Help Prevent Unauthorized Account Removals from AWS Organizations](3.1-Blog1/)

This blog discusses security recommendations published by the AWS Customer Incident Response Team (AWS CIRT). The article explains how attackers may attempt to remove compromised accounts from AWS Organizations in order to bypass centralized security controls. It also presents best practices such as implementing Service Control Policies (SCPs), applying the principle of least privilege, securing root accounts, and monitoring CloudTrail events.

### [Blog 2 - Welcoming the AWS Customer Incident Response Team](3.2-Blog2/)

This blog introduces the AWS Customer Incident Response Team (AWS CIRT), a specialized team that helps customers prepare for, respond to, and recover from cybersecurity incidents in AWS environments. The article highlights AWS incident response services, security best practices, and the importance of proactive security planning in cloud infrastructure.

### [Blog 3 - Amazon Redshift Delivers Faster Performance for BI Dashboards and Real-Time Analytics](3.3-Blog3/)

This blog presents the latest performance improvements in Amazon Redshift. The enhancements help organizations accelerate Business Intelligence (BI) dashboards, improve query execution times, support real-time analytics workloads, and optimize resource utilization. The article demonstrates how AWS continuously improves data analytics services to support modern enterprise requirements.

Through researching and sharing these blogs, I gained knowledge in:
